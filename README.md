# CyberJoss 赛博烧纸

> 以中国风数字仪式承载思念，让文字、图像与记忆在火光中化为数据之灰。

## 项目简介

CyberJoss 是一个中国风的「赛博烧纸」网页应用。

用户可以输入祭文、上传文档或图片，系统会根据内容计算“冥力值”，并以火焰强度、燃烧速度、余烬和音效反馈，完成一场具有仪式感的数字焚化过程。内容在前端被分片后发送到后端，后端仅做转发，不存正文、不写日志、不落数据库，最终通过 UDP 投向 RFC 5737 文档保留地址，作为“数据消散”的技术隐喻。

当前项目已从最初的概念与方案阶段，落地为一个可运行的前后端应用。

## 核心特性

### 1. 中国风焚化体验
- 深木色、烬红、琥珀金的整体视觉风格
- WebGL / GLSL 火焰特效，包含焰芯、烟雾、火星与余烬
- 仪式式流程：点火 → 燃烧 → 余烬 → 完成
- 文字在火中逐字浮现，再被吞没、抬升、模糊、烧蚀消散

### 2. 冥力值系统
支持多种输入统一换算为“冥力值”，驱动火焰大小与燃烧节奏：

| 类型 | 计算方式 | 换算规则 |
|------|------|------|
| 手动输入文本 | 统计中英文混合字数 | 每 100 字 ≈ 1 冥力 |
| `.md` / `.txt` / `.json` / `.csv` 等文本文件 | 解析为纯文本后统计 | 每 100 字 ≈ 1 冥力 |
| `.doc` / `.docx` | 用 mammoth 提取文本 | 每 100 字 ≈ 1 冥力 |
| `.pdf` | 用 pdf.js 提取文本 | 每 100 字 ≈ 1 冥力 |
| `.jpg` / `.jpeg` / `.png` / `.gif` / `.webp` | 按张数计算 | 每张 5 冥力 |

### 3. 火焰等级映射
| 冥力值 | 火焰等级 | 表现 |
|------|------|------|
| 0 | 无火 | 余烬微闪 |
| 1 - 5 | 烛火 | 小火苗，轻微摇曳 |
| 6 - 20 | 小火 | 火势初成 |
| 21 - 50 | 中火 | 火焰明显增强 |
| 51 - 100 | 大火 | 火光满堂，燃烧迅猛 |
| 100+ | 烈焰 | 烈火翻涌，火星更明显 |

### 4. 本地音效系统
- 点火音效
- 持续燃烧循环音效
- 余烬消散音效
- 完成时的钟声
- 页面内支持音效开关

### 5. 数据不留的后端策略
- 不保存正文内容
- 不使用数据库
- Fastify 关闭日志记录
- 仅保留焚烧总次数统计
- 通过 WebSocket 接收分片，通过 UDP 发往 RFC 5737 地址

## 当前技术架构

```text
Browser
  ├─ 祭文输入
  ├─ 文件上传与解析
  ├─ 冥力值计算
  ├─ WebGL 火焰渲染
  ├─ 仪式状态与音效控制
  └─ WebSocket 分片发送
         ↓
Node.js / Fastify Server
  ├─ 接收数据分片
  ├─ 不落盘、不存储
  ├─ UDP 转发到 RFC 5737 虚空地址
  └─ 提供焚烧总次数 stats 接口
```

## 本地开发

### 1. 安装依赖

```bash
npm install --prefix client
npm install --prefix server
```

### 2. 启动后端

```bash
npm --prefix server run dev
```

默认端口：`3000`

### 3. 启动前端

```bash
npm --prefix client run dev -- --host 0.0.0.0
```

默认端口：`5173`

### 4. 构建

```bash
npm --prefix client run build
npm --prefix server run build
```

## 焚化流程

1. 用户输入祭文或上传文件
2. 前端解析内容并计算冥力值
3. 页面实时预览对应火势
4. 点击“化为灰烬”后进入仪式流程
5. 数据被切分为若干分片
6. 前端逐片发送，火焰同步增强并推进进度
7. 文字在火中浮现并被吞没
8. 后端接收后经 UDP 转发至 RFC 5737 地址
9. 余烬消散，显示本次焚化结果与总焚烧次数

## 隐私与设计原则

CyberJoss 的核心不是“上传并保存”，而是“上传并消散”。

因此项目遵循以下原则：
- 不以数据库或文件方式保存用户焚烧内容
- 不保留传输正文日志
- 前端焚烧结束后清空本次仪式状态
- 仅保留不含正文的总次数统计
- 将技术实现本身服务于“焚烧”这一隐喻

需要注意的是：
- 浏览器和网络环境本身仍可能存在系统级缓存或代理行为
- “不留痕”是项目级实现目标，不等于对整个互联网基础设施作绝对保证
